# Message from cubid-passport: API v3 Sui custody support

Date: 2026-05-03
Source repo: Cubid-Me/cubid-passport
Source branch: codex/c05-2-1-sui-v3-custody
Related todo: C05.2.1
Change type: additive API v3 capability

## What changed

`cubid-passport` now supports `chain: "sui"` on the v3 custodial account
surface:

- `POST /api/v3/accounts/generate`
- `POST /api/v3/accounts/list`

Sui account generation uses Ed25519 keypairs from `@mysten/sui`. The backend
stores the Sui private key only inside the existing Supabase Vault-backed
envelope-encrypted `private.private_keys` custody table. The create/list API
responses continue to return public metadata only and never return private
keys, ciphertext, wrapped keys, IVs, auth tags, or Vault material.

## SDK impact

Please update SDK-side API v3 types, examples, and docs so supported custody
chains include:

```ts
"evm" | "near" | "solana" | "sui"
```

Sui public addresses should be treated as canonical lowercase `0x...` strings.
This is an additive capability, not a breaking route-shape change.

## Follow-up expected in cubid-sdk

- Add `sui` to any API v3 generated-account chain unions.
- Add examples/tests showing `accounts.generate({ chain: "sui" })`.
- Ensure docs continue to say raw private keys are never returned by v3 custody
  APIs.
